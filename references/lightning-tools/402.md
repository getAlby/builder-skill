# HTTP 402 Payment Required

## Overview

`fetch402` is a drop-in replacement for `fetch` that automatically handles HTTP 402 Payment Required responses. When a server responds with a 402 status and a payment challenge, the function pays the Lightning invoice using your wallet and retries the request with proof of payment — all transparently.

It works with multiple 402 protocols: L402, x402, and MPP (lightning-charge).

## fetch402

```typescript
import { fetch402 } from "@getalby/lightning-tools";
import { nwc } from "@getalby/sdk";

const client = new nwc.NWCClient({
  nostrWalletConnectUrl: "nostr+walletconnect://...",
});

// The NWC client implements the Wallet interface directly
const response = await fetch402(
  "https://paid-endpoint.example.com",
  {}, // standard fetch options (method, headers, body, etc.)
  { wallet: client }
);

const body = await response.text();
console.log(body);
```

### Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `url` | `string` | The URL to fetch |
| `fetchArgs` | `RequestInit` | Standard fetch options |
| `options.wallet` | `Wallet` | Wallet that can pay invoices (NWC client works directly) |
| `options.maxAmount` | `number?` | Optional maximum amount in sats the wallet is allowed to pay |

### Wallet Interface

```typescript
interface Wallet {
  payInvoice(args: { invoice: string }): Promise<{ preimage: string }>;
}
```

NWC clients from `@getalby/sdk` implement this interface directly — no adapter needed.
